═══════════════════════════════════════════════════════
SECTION 1 — ONBOARDING & PROFILE
═══════════════════════════════════════════════════════

[✅] 1.1 — Onboarding Survey
  What: 4-step survey collecting name, experience, focus areas, goal
  How: Multi-step form at /onboarding
  AI: Claude analyzes results → generates personalized profile (greeting,
      tagline, coaching style, weekly goal, top skills, first challenge,
      personalized insights)
  Storage: localStorage under "speakup_user"
  Files: src/app/onboarding/page.tsx
         src/app/api/onboarding/route.ts

[✅] 1.2 — Personalized Dashboard
  What: Dashboard that reads AI profile and shows personalized content
  How: Reads localStorage profile → shows focus area cards, coach message,
       weekly goal, skills to build, first challenge, stats
  Files: src/components/Dashboard.tsx
         src/app/dashboard/page.tsx

[❌] 1.3 — User Accounts & Authentication
  What: Real accounts so users keep progress across devices
  Why: Currently everything lives in localStorage — lost if browser cleared
  How to build:
    - Use Clerk (fastest) or Supabase Auth
    - Replace localStorage with database calls
    - Onboarding saves to DB instead of localStorage
    - Progress (XP, sessions, weaknesses) tied to user ID
  Needs: clerk or @supabase/supabase-js package
  Priority: HIGH — required for paid version

SECTION 2 — DEBATE AI
═══════════════════════════════════════════════════════

[✅] 2.1 — Debate Hub (3 Levels)
  What: Level selector showing Foundations, Strategy, Competition
  How: Recommends level based on onboarding experience
  Files: src/components/DebateHub.tsx, src/app/debate/page.tsx

[✅] 2.2 — Progression System
  What: AI decides whether student advances, stays, or drops between levels
  How: XP earned per session → at 100% Claude evaluates → advance/stay/drop
  XP rates: Sparring=25, Speech=18, Rebuttal=12, Case Builder=10
  Level rules: Beginner starts L1, Intermediate L2, Advanced L3
               Drop to L1 only if fundamental misunderstanding shown
  Files: src/lib/progress.ts
         src/components/debate/ProgressBar.tsx
         src/app/api/debate/progress/route.ts

[✅] 2.3 — Level 1: Foundations Lessons
  What: 8 lessons — debate types, CWI model, Aff/Neg + Pro/Con,
        flowing, cross-ex, rebuttals, round structure (PF/LD/Policy), practice
  Files: src/components/debate/Level1.tsx

[✅] 2.4 — Level 2: Strategy Lessons
  What: 8 lessons — turns, offensive/defensive strategy, reading the flow,
        advanced CX, block files, time allocation, off-case, practice
  Files: src/components/debate/Level2.tsx

[✅] 2.5 — Level 3: Competition Lessons
  What: 7 lessons — tournament structure, judge adaptation, theory,
        K debate, speed/clarity, elim pressure, post-round analysis
  Files: src/components/debate/Level3.tsx

[✅] 2.6 — AI Sparring Partner
  What: Real-time debate against AI opponent
  How: Chat interface — AI plays opposition, fires POIs, attacks weak points
       Level-aware: friendly (L1), strategic (L2), relentless (L3)
       After 3 exchanges, student can request Judge Ballot
  Files: src/components/debate/SparringPartner.tsx
         src/app/api/debate/sparring/route.ts

[✅] 2.7 — Speech Upload + AI Feedback (6 areas)
  What: Student writes speech → gets feedback across 6 dimensions
  Areas: Structure, Argument, Evidence, Rebuttal, Impact, Delivery
  Level-aware: beginner lens / intermediate standard / competition standard
  Files: src/components/debate/SpeechFeedback.tsx
         src/app/api/debate/speech-feedback/route.ts

[✅] 2.8 — Weakness Detector
  What: Tracks weakness patterns across all sessions
  How: After each session, weakness label saved → aggregated over time
       Shows top 3 weaknesses + specific drills to fix each
  Files: src/components/debate/WeaknessDetector.tsx
         (data stored via src/lib/progress.ts)

[✅] 2.9 — Rebuttal Trainer
  What: Timed challenge — AI gives argument, student rebuts in 90 seconds
  How: 3 rebuttal inputs, countdown timer, AI grades each rebuttal
       Level-aware topics and grading standards
  Files: src/components/debate/RebuttalTrainer.tsx
         src/app/api/debate/rebuttal/route.ts

[✅] 2.10 — Case Builder
  What: Student enters motion + side → AI builds complete debate case
  Output: Framing, Mechanism, 2-3 Arguments, Examples, Impacts,
          Weighing, Opening Speech Script
  Level-aware: simple (L1), strategic (L2), competition-level (L3)
  Files: src/components/debate/CaseBuilder.tsx
         src/app/api/debate/case-builder/route.ts

[✅] 2.11 — AI Judge Mode
  What: After sparring session, AI renders a judge ballot
  Output: Winner, Score/100, Reasoning, Clash Analysis,
          Impact Weighing, Main Improvement
  Level-aware: beginner criteria / intermediate / competition standard
  Files: (built into SparringPartner.tsx)
         src/app/api/debate/judge/route.ts

[✅] 2.12 — Debate Arena (Tabbed Shell)
  What: Unified tabbed interface wrapping all debate features per level
  Tabs: Lessons | Sparring | Rebuttal | Case Builder | Speech | Weaknesses
  Files: src/components/debate/DebateArena.tsx

  What: Persist user profiles, progress, and session history
  Tool: Supabase (PostgreSQL — works well with Next.js + Clerk)
  Tables needed:
    - users (id, clerk_id, name, experience, focus, goal, profile_json)
    - debate_progress (user_id, level, xp, sessions_json, weaknesses_json)
    - speaking_progress (user_id, level, xp, sessions_json, weaknesses_json)
  Files needed: src/lib/db.ts
                src/app/api/user/route.ts
  Priority: REQUIRED for paid version

[❌] 6.3 — Stripe Payments
  What: Subscription billing for premium access
  Tool: Stripe + stripe npm package
  Pricing model discussed:
    - Free: Onboarding + 3 AI feedback sessions/month
    - Pro ($9.99/mo): Unlimited feedback across all sections
    - School ($29.99/mo): Up to 30 students + teacher dashboard
  What to build:
    - /pricing page
    - Stripe checkout session API
    - Webhook to update user subscription in DB
    - Usage tracking per user (count API calls)
    - Paywall middleware: check subscription before AI routes
  Files needed: src/app/pricing/page.tsx
                src/app/api/stripe/checkout/route.ts
                src/app/api/stripe/webhook/route.ts
                src/lib/stripe.ts
                src/middleware.ts (update to check subscription)
  Priority: REQUIRED for paid version

  What: Free users get limited AI sessions, paid users get unlimited

       If over limit → redirect to /pricing
  Files needed: update all /api/debate/*, /api/public-speaking/*,
